# CAPTURE AND ANALYZE NETWORK TRAFFIC USING WIRESHARK

## Objective:
The objective of this task is to capture live network traffic using Wireshark and analyze the captured packets to identify different network protocols and traffic types.

## Tools Used:
- Wireshark
- Windows Operating System
- Internet Connection

## Procedure:

1. Installed and launched Wireshark.
2. Selected the active network interface (Wi-Fi/Ethernet).
3. Started packet capture.
4. Generated network traffic by browsing websites and sending ping requests.
5. Captured traffic for approximately one minute.
6. Applied protocol filters to analyze captured packets.
7. Identified multiple network protocols.
8. Saved the capture file in `.pcapng` format.

## Protocols Identified:

### DNS (Domain Name System)
DNS packets were observed during domain name resolution requests and responses.

### TCP (Transmission Control Protocol)
TCP packets were identified as part of reliable communication between client and server systems.

### TLS (Transport Layer Security)
TLS packets were detected during secure web browsing activities, providing encrypted communication.

## Packet Analysis Summary

| Protocol | Purpose |
|-----------|----------|
| DNS | Resolves domain names to IP addresses |
| TCP | Provides reliable data transmission |
| TLS | Secures network communication through encryption |

## Outcome:

Successfully captured and analyzed live network traffic using Wireshark.
Multiple protocols including DNS, TCP, and TLS were identified and examined. 
This task improved understanding of packet capture, protocol analysis, and network troubleshooting concepts.

## Conclusion:

Wireshark proved to be an effective network protocol analyzer for monitoring and examining network communications. 
Through this task, basic packet analysis skills and protocol awareness were developed.
